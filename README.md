# KR-DSS · 국내 전자서명 규격 및 신뢰검증 기술 실증 연구

> 단순 본인인증을 넘어, **전자문서 자체에 결합되는 전자서명**과 그 신뢰를 **시스템적으로 검증**하는 체계를 실증한다.

- **사업명**: 국내 전자서명 규격 및 신뢰검증 기술 실증 연구
- **발주기관**: 한국인터넷진흥원(KISA) 디지털서명인증팀
- **수행사**: 주식회사 일렉서티(ELECTCERTI) · PM 김용성 대표이사
- **사업기간**: 계약체결일 ~ 2026.12.10
- **부제**: KR-AdES · KR-TL · KR-DSS 기반 국가 전자서명 신뢰 인프라 실증

---

## 1. 배경 · 왜 지금인가

전자서명법 전부개정(2020) 이후 6년, 간편인증 등 **인증 수단은 다양해졌으나** 그 활용은 대부분 로그인·본인확인 등 **인증형**에 머물러 있다. 전자문서 자체에 서명이 결합되어 진위확인·부인방지를 보장하고 문서 단독으로 독립 검증되는 **전자문서 전자서명**은 여전히 표준화 공백 상태다.

| 구분 | 인증형 전자서명 | 전자문서 전자서명 *(본 사업 대상)* |
| --- | --- | --- |
| 목적 | 신원확인 중심 | 진위확인·부인방지 |
| 구조 | 정보·서명 분리 | 문서에 서명 결합 |
| 검증 | API 검증 | 문서 독립 검증 |
| 용도 | 로그인·본인확인 | 전자계약·공공·민간 문서 |

### 해소하고자 하는 3대 공백
1. **서명규격 표준의 공백** → `KR-AdES`
2. **신뢰검증 체계의 공백** → `KR-TL`
3. **상호운용 기반의 공백** → `KR-DSS`

---

## 2. 3대 산출물

세 산출물 모두 **ETSI EN 319 1xx / eIDAS** 국제표준에 완전 정합(Full Compatibility)하도록 설계한다.

### 🔹 KR-AdES — 전자서명 규격 표준화 (과업 1)
6종 포맷 공통 **Core 모델**을 상속한 프로파일과 **B/T/LT/LTA** 레벨로 전자문서 서명 규격을 표준화한다.

- **형식(6종)**: KR-XAdES(XML) · KR-CAdES(CMS) · KR-PAdES(PDF) · KR-JAdES(JSON) · **KR-HAdES(HWP)** · **KR-MAdES(Markdown)**
- **레벨**: KR-B(기본) / KR-T(시점보증) / KR-LT(장기검증) / KR-LTA(장기보존)
- **패키징**: Enveloped / Enveloping / Detached / ASiC
- **정합 표준**: EN 319 122·132·142 / TS 119 182 / EN 319 162(ASiC) / EN 319 102(검증)
- **구조**: `KR-AdES → KR-AdES-Core(공통 데이터 모델·규칙) → 6종 프로파일` 상속 구조로 확장성과 설계 일관성을 동시 확보
  - Core 필드: `profileId · documentInfo · signedObject · signatureInfo · timestampInfo · packagingInfo · extensionInfo` (JSON Schema / XML Schema 기반)

> 국내에서 많이 쓰는 **HWP**와 차세대 문서인 **Markdown**까지 하나의 체계에 담는 것이 차별점.

### 🔹 KR-TL — 신뢰검증 체계 (과업 2)
인정사업자 인증서를 **자동 수집·동기화·배포**해 검증 가능한 한국형 신뢰목록(Trust List)을 구축한다.

- **운영 5단계**: ① 신뢰정보 제출(인정사업자) → ② 수집·검토(KISA) → ③ KR-TL 생성·전자서명 → ④ TL 갱신(이용기관) → ⑤ 검증·활용
- **핵심 차별점**: 인정평가 결과를 KR-TL 반영 기준 정보로 활용 → **제도적 인정 + 기술적 검증을 단일 흐름으로 연결**
- **스키마**: `KR-TrustList → SchemeInformation / TrustServiceProviderList → TSP → TSPInformation·TSPServices → ServiceInformation / HistoryInformation`
  - 핵심 6영역: Scheme Information / TSP Information / Service Information / Service Digital Identity / Extensions / Signature
- **정합 표준**: ETSI TS 119 612 (EU TL 골격 + 국내 인정제도 반영 확장 필드)
- 신뢰목록 자체에 전자서명을 부여해 배포본의 무결성·진본성 보장

### 🔹 KR-DSS — 전 과정 통합 지원 SDK (과업 3)
생성·추출·검증·신뢰연동을 아우르는 SDK와 **ETSI 표준 검증보고서**로 전자서명 전 과정을 통합 지원한다.

- **공통 API**: `Sign()` / `Extract()` / `Verify()` / `GetTrustList()` / `RefreshTrustList()` / `Timestamp()`
- **계층**: 공통 인터페이스 / 핵심 서비스 / 포맷 어댑터(6종) / 보안·크립토 / 플랫폼
- **검증보고서 4종**: Diagnostic Data / **ETSI Validation Report**(EN 319 102-1·TS 119 102-2) / Detailed Report / Simple Report
- 웹·서버 환경 표준 모듈로 구현, 문서편집기·뷰어·기안기·서명앱 등 다양한 이용환경에서 공통 인터페이스로 호출

## 3. 전자서명 전 과정 실증 (PoC)

생성 → 패키징 → 추출 → KR-TL 조회 → 검증 → 결과반환 전 구간을 실증 시스템에서 통합 검증한다.

```
[A. 생성]  전자문서 → 서명 생성 → 패키징·문서 주입 → 서명문서
[B. 검증]  추출 → KR-TL 조회 → 인증서·정책 검증 → 무결성 검증 → 결과·보고서
```

- **판정 3종**: 유효(`TOTAL_PASSED`) / 확인불가(`INDETERMINATE`) / 무효(`TOTAL_FAILED`)
- **실증 구성**: 가상 인정사업자 · KISA/KR-TL 관리 · 표준 SDK · 가상 이용사 서비스 (독립 테스트베드)

---

## 4. 추진 전략

```
1 분석(Gap→요구사항) → 2 설계(KR-* 표준안) → 3 구현(KR-DSS) → 4 검증(PoC·결과보고)
```

- 결과 환류·확산: 정책·제도 수립 / 표준화 제안 / 산업 확산 지원 / 국제 상호인정 / 후속 연구
- **실증 대상 포맷·범위는 RFP에 따라 KISA와 협의하여 핵심 포맷(PDF·HWP)부터 단계적으로 확정**하며, 나머지는 공통 Core 상속으로 설계 일관성만 확보

## 5. 기대효과

| 산출물 | 활용 방향 |
| --- | --- |
| KR-AdES | 원격전자서명 표준화 → 상호운용성 확보 |
| KR-TL | 원격전자서명 검증체계 → 인증서 통합 검증 |
| KR-DSS | 원격전자서명 서비스 → 전자계약·공공·민간 확대 |

- 인정평가 제도 고도화 / 장기검증 기반 법적 증거력 강화 / eIDAS 등 국제 상호운용성 확보
- **추천 1순위: 원격전자서명** (2025 전자서명포럼 발표 시 사업자 수요 최상, eIDAS 원격서명(QSCD) 흐름과 정합)

## 6. 수행사 · 전문성

주식회사 일렉서티는 2023.07 설립 이후 전 사업이 전자서명·디지털인증 신뢰체계로 일관된다.

- **전문성 4축**: 국제표준 분석·구현 / 전자서명 기술 구조 / 산·학·연 연구반 / AI 전환 확장
- **국내 기술표준 직접 집필**: 간편인증 가이드라인(2023) · 전자여권 가이드라인(2024) · 본인확인 API 규격(2025)
- **주요 연혁**: 전자서명 상호연동 연구반(2023~) → 전자여권·재외동포(2024~) → 국제표준 연구반(2025~) → 참조모델·TTA 표준화(2025~) → 본 사업·원격전자서명(2026~)
